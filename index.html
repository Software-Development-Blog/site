What is the Difference between an Overloading and an Overriding?

Method overloading is a form of static, or compile-time, polymorphism. It allows you to change method logic, list of arguments and return type, but remain the same method name. Access modifiers for overloaded methods can be the same, more restrictive or less restrictive.

Method overriding is a form of a dynamic, or run-time, polymorphism. Methods in a subclass must have the same name, list of arguments or their subtypes, and return type or its subtype. The access level of the overridden method must be the same or less strict than the access level of the superclass method.

What is the Difference between a Heap and a Stack Memory?

Heap is used for dynamic objects with unpredictable lifetyme. Heap memory is generally larger than stack memory. Access to a heap memory is slower. Heap memory is typically shared among all threads.

Stack memory is used to store method call frames, local variables, and control flow information. Stack uses LIFO data flow, so the most recently called method is on the top of a stack. Typically each thread has its own stack.

Explain an Output of the Code

System.out.println(1 + 2 + “Text”); // 3Text

System.out.println(“Text” + 1 + 2); // Text12

Java evaluates expressions from left to right. During concatenation, Java retrieves string representations of the integers.

What is the Difference between a Shallow and a Deep Copy?

Shallow copy is a separate entity that references the nested objects contained by the original object. Changes made to the nested objects may affect original contained objects. Typically shallow copy is created using a copy constructor or “clone” method. Copy constructor takes an instance of a class as its argument, and creates a new instance with the same state. Deep copy duplicates an original object recursively. Changes of a deep copy object doesn’t affect an original object.

What is a Garbage Collector and How Does It Work?

GC is a component of the JVM responsible for automatic memory management. Short-time objects (which quickly become unreachable) are more likely to be removed by GB. While long-lived objects have less probability to be removed.

What is the Difference Between a Constructor and a Method?

Constructors are used to initialize objects when they are created. Constructors must have the same name as their class. Constructors are called automatically when you create an instance by a “new” keyword, while methods are called by “.” dot operator. Constructor don’t have a return type while methods have (even void). Constructors and methods can be overloaded, and can have different access modifiers.

TYPE SYSTEM

STRONGLY AND WEAKLY TYPED SYSTEMS. There are distinguished strongly and weakly typed programming languages. In strongly typed systems type errors are detected during a compilation process, while in weakly typed systems type errors might not be checked until a run-time. Strongly typed languages enforce explicit type promotion (type casting). Implicit type promotion (type coercion) might be done automatically for narrowing primitive types conversion during operations over those types. In Java type casting can be done by using a target type keyword enclosed in round braces: “int newType = (int) oldType”.

AUTOBOXING

Autoboxing implies automatic conversion of primitive data types to their corresponding wrapper classes. Unboxing represents a reverse conversion - from wrapper classes to their corresponding primitive values. Autoboxing is commonly used when working with collections like lists, sets, maps, which require objects instead of primitive types.

COMPOUND DATA TYPES

Compound data types can hold multiple primitive or compound types as a single unit. Compound data types include Arrays, Enums, Collections Framework, Data structures, User-defined data types (classes), and Generics (parameterized types).

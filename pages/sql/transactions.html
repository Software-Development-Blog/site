Database Transactions

Transaction is a sequence of one or more database operations treated as a single unit. Transactions share ACID properties - Atomicity, Consistency, Isolation, and Durability.

Atomicity. Guarantees that a transaction is treated as a single, indivisible unit of work. Either all the changes within a transaction are committed to the database, or, if any part of the transaction fails, the entire transaction is rolled back to its previous state.

Consistency. Ensures that a transaction brings the database from one consistent state to another, it maintains the validity and integrity of data.

Isolation. Ensures that multiple transactions can occur concurrently without interfering with each other.

Durability. Guarantees that once a transaction is committed, its changes are permanent.

Data Anomalies

Dirty Read. Occurs when a transaction reads uncommitted changes made by another transaction that are later rolled back.

Non-Repeatable Read. Occurs when a transaction re-reads data and obtains different results due to other committed transactions modifying the data.

Phantom Read. Occurs when a transaction re-executes a query and obtains different rows due to other committed transactions inserting or deleting the rows.

Serialization Anomaly. Occurs when concurrent transactions, individually correct in isolation, produce incorrect results when executed together. It violates the isolation principle.

Transaction Isolation Levels

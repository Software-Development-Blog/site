Spring Technical Interview

What is the Difference between Spring Framework and Java EE?

Java EE is a set of specifications like Java Servlet API, Java Persistence API, Java Message Service, Java Beans, Bean Validation, and others to build enterprise-grade applications. Spring Framework is a stand-alone framework that may use Java EE specifications but in general Spring provides its own way to build enterprise applications.

Spring promotes the use of POJOs (Plain Old Java Objects), encourages the use of dependency injection, aspect-oriented programming, and loose coupling, which are not specific to Java EE. Spring has its own set of modules and components.

Java EE and Spring have different communities and maintainers, Oracle and VMware respectively.

What Spring Does Include?

The Spring ecosystem contains many projects like Spring Framework, Spring Boot, Spring Data, Spring Security, Spring Cloud, Spring WebFlux, and others.

What are Spring Components?

Core Container (Beans, Core, Context, and Expression Language), Testing (unit and end-to-end tests), Data access (ORM, Transactions, JDBC), Web (Servlets, Struts), and AOP.

What is the Difference between Spring Framework and Spring Boot?

Spring Boot introduces the concept of "starter" dependencies, which include predefined sets of dependencies to address common use cases. Spring Boot includes embedded web servers (e.g., Tomcat, Jetty, Undertow) by default.
